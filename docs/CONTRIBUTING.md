Thank you for your interest in contributing to this project!

Guidelines
- Create an issue for any bug or feature request before starting work.
- Fork the repository, create a feature branch, and open a pull request against the `frontend` branch.
- Keep changes focused and small. Add tests for new functionality where reasonable.
- Follow existing code style and run linters/tests locally before opening the PR.

Commit message style
- Use present-tense, short subject lines and a longer body if needed.

Code of Conduct
Please follow the project's Code of Conduct (CODE_OF_CONDUCT.md).
